# Events

The Events screen shows a live stream of [events](../core/events/) — the triggers that cause your tasks to run. Use it to monitor your automations and investigate issues.

## Event list

<figure><img src="../.gitbook/assets/events_mechanic.jpg" alt="The Events list showing recent events with their topics and task run results"><figcaption></figcaption></figure>

Events are displayed in reverse chronological order, showing the time, topic, and task run results for each event. The list updates in real time as new events arrive.

### Filtering

Use the **Filters** button to narrow results by search text, date range, error type, event topic, or specific task. The **Errors** tab provides quick access to all events where something went wrong.

Save filter combinations as **custom views** for quick access.

## Event detail

<figure><img src="../.gitbook/assets/event_detail_mechanic.jpg" alt="An event detail view showing event runs, task runs, action runs, logs, and event data"><figcaption></figcaption></figure>

Click any event to see its full details — the event run with its task runs and action runs, the raw event data (JSON), and any parent or child events.

You can **retry** a failed event run, **cancel** a pending one, or **export** the event and all related runs as JSON via email.
